# Signal Foundry

**Signal Foundry** is a Streamlit-based unstructured text analytics app for turning messy text sources into fast, inspectable signals: term frequencies, keyphrases, sticky concepts, entity-like names, topic clusters, network graphs, sentiment estimates, and optional AI-generated interpretation.

The app is designed for two operating modes:

1. **Interactive app mode** — upload files, paste URLs, or paste text directly into the Streamlit UI.
2. **Offline harvester mode** — preprocess large or sensitive datasets locally with `harvester.py`, then upload the resulting privacy-preserving JSON sketch into the app.

Signal Foundry is especially useful for rapid analysis of transcripts, research corpora, support exports, meeting logs, articles, reports, PDFs, slide decks, and spreadsheet-based text datasets.

---

## Features

### Supported input sources

Signal Foundry can process:

- CSV files
- Excel workbooks (`.xlsx`, `.xlsm` depending on deployment support)
- WebVTT transcript files (`.vtt`)
- Plain text files (`.txt`)
- JSON offline sketches generated by `harvester.py`
- PDFs
- PowerPoint files (`.pptx`)
- Web pages via URL paste
- Manual raw text pasted into the sidebar

### Core analytics

The app produces several layers of analysis from the input corpus:

- **Word cloud** for rapid visual orientation
- **Frequency tables** for top terms and phrases
- **TF-IDF keyphrases** to identify distinctive vocabulary
- **NPMI sticky concepts** to surface statistically meaningful two-word phrases
- **Entity-like extraction** for important capitalized names and terms
- **Temporal trends** when a date column is available
- **Category-aware analysis** when a category column is selected
- **Network graph** showing co-occurrence and contextual relationships
- **Topic discovery** using topic modeling when enough document-level text is available
- **Bayesian sentiment inference** based on positive and negative term evidence
- **Maturity assessment** based on configurable linguistic markers
- **Hybrid signature / QR artifact** for document-grounding and chain-of-custody style workflows

### Optional AI Analyst

Signal Foundry can optionally call an OpenAI-compatible chat-completions endpoint to interpret the statistical sketch produced by the app.

The AI Analyst is intended to analyze derived metadata such as frequencies, clusters, and graph signals. It is not required for the core analytics workflow.

A typical deployment contains:


## Configuration

Most analysis options are configured from the Streamlit sidebar.

### Data input

Use the sidebar to:

- Upload one or more files
- Paste one or more URLs
- Paste raw text manually
- Upload an offline JSON sketch generated by `harvester.py`
- Reset all analysis state
- Choose whether a new scan should clear existing results or add to the current corpus

### Cleaning options

The app includes controls for:

- Removing chat artifacts
- Removing HTML
- Removing URLs
- Unescaping HTML entities
- Custom stopwords
- Removing prepositions
- Keeping or removing hyphens
- Keeping or removing apostrophes

### Processing options

The app includes controls for:

- Minimum word length
- Dropping integers
- Bigram computation
- Optional lemmatization
- Number of top terms to display
- Word cloud size and appearance
- Graph thresholds
- Topic and sentiment views, where supported by available dependencies and input data

The exact secret names should match the names expected in `mainapp.py`.

## Usage

### Path A: Interactive scan

1. Launch the app with `streamlit run mainapp.py`.
2. Upload files in the sidebar, or paste URLs/text.
3. Configure text, date, and category columns when prompted.
4. Run the scan.
5. Review the dashboard outputs: word cloud, frequency tables, keyphrases, sticky concepts, graph, topics, sentiment, and maturity assessment.

### Path B: Column-controlled spreadsheet scan

For CSV and Excel files, expand the file-specific configuration panel when you need more control.

Use this mode when:

- The text is not in the first column
- You need a date column for time-travel filtering
- You need a category column for segmentation
- The file has multiple candidate text fields

### Path C: Offline harvester workflow

For very large or sensitive datasets, use `harvester.py` outside the Streamlit app.

Example:

```bash
python harvester.py --input data.csv --col text --output sketch.json
```

Then upload `sketch.json` into the app under **Load Offline Analysis (Harvester)**.

Offline sketches are useful when raw text should not leave a secure environment. They preserve aggregate statistics such as counts and bigrams, while omitting raw document text.

Current offline sketch limitations:

- Topic modeling may be unavailable because raw topic documents are not stored.
- Entity extraction may be unavailable unless entity counts are included in the sketch.
- Time-series views may be unavailable unless temporal counts are included in the sketch.

## Offline harvester

`harvester.py` is a command-line companion script for preprocessing large datasets.

### Basic command

```bash
python harvester.py --input data.csv --col text --output sketch.json
```

### Arguments

| Argument | Description |
|---|---|
| `--input` | Path to an input CSV, Excel file, or folder of supported files |
| `--col` | Name of the text column to analyze |
| `--output` | Output JSON sketch path; defaults to `sketch.json` |
| `--chunksize` | Number of rows per chunk; defaults to `50000` |
| `--no-bigrams` | Disable bigram calculation |

### Recommended workflow

1. Run the harvester on the secure machine where the data lives.
2. Inspect the terminal output for row counts and vocabulary size.
3. Upload the generated sketch JSON into Signal Foundry.
4. Use the app dashboard to explore the aggregate results.

## Interpreting the dashboard

### Word cloud and top terms

Use this for a quick sense of what dominates the corpus. If the cloud is full of obvious noise, add those terms to the stopword list and rescan.

### TF-IDF keyphrases

Use this to find terms that are distinctive to the corpus or selected slice. These are often more useful than raw frequency when looking for domain-specific vocabulary.

### NPMI sticky concepts

Use this to find phrase pairs that occur together more meaningfully than chance. This is useful for finding terms of art, recurring concepts, and tightly coupled ideas.

### Entities

Use this to identify important names, organizations, acronyms, products, or capitalized terms. Entity-like extraction is heuristic, so treat it as a signal-finding tool rather than a named-entity recognition guarantee.

### Network graph

Use this to understand relationships among terms. If the graph is too dense, raise the minimum link frequency. If it is too sparse, lower the threshold.

### Topic discovery

Use this to identify recurring thematic clusters. Topic modeling works best when the app has enough document-level text available.

### Bayesian sentiment

Use this as a lightweight directional estimate based on positive and negative term evidence. It is more useful for comparing slices than for claiming an absolute emotional truth about a corpus.

### Maturity assessment

Use this to estimate organizational or operational maturity based on configured linguistic markers. Treat it as an interpretive model, not as a final score without human review.

## Deployment

### Streamlit Community Cloud

1. Push the repository to GitHub.
2. Create a Streamlit Community Cloud app pointing to `mainapp.py`.
3. Add required secrets in the Streamlit app settings.
4. Confirm `requirements.txt` includes all dependencies.
5. Reboot the app after changing dependencies or secrets.

### Local deployment

```bash
streamlit run mainapp.py
```

### Private/server deployment

For private deployments, consider:

- Running behind an authentication layer
- Restricting arbitrary URL fetches if deployed in a sensitive network
- Keeping AI Analyst disabled unless keys and data-handling expectations are clear
- Running `harvester.py` close to sensitive data and only moving sketch JSON files into the app

## Security and privacy notes

- Offline harvester sketches are designed to avoid storing raw document text.
- URL ingestion should be treated carefully in production because the app fetches remote content.
- Do not store API keys or app passwords in the repository.
- Use Streamlit secrets or your deployment platform’s secret manager.
- If the app is exposed publicly, consider adding authentication and limiting accepted input types.
- The AI Analyst should be understood as optional. Review the app’s current prompt construction before using it with sensitive corpora.

## Troubleshooting

### The app starts, but some features are unavailable

Check the dependency status panel in the sidebar. Some features depend on optional libraries such as `nltk`, `scikit-learn`, `scipy`, `qrcode`, PDF parsing, or PowerPoint parsing packages.

### Excel results look wrong

Confirm that the correct text column was selected. For multi-column spreadsheets, avoid relying on the first column unless it is definitely the intended text field.

### The graph is unreadable

Raise the minimum link frequency or reduce the number of top terms displayed.

### The graph is empty

Lower the minimum link frequency, enable bigrams, or scan more text.

### Topic modeling is unavailable

Topic modeling requires enough document-level text. It may not be available for offline sketches that intentionally omit raw topic documents.

### AI Analyst returns an error

Confirm that the API key, base URL, and model name are configured correctly. The app’s core analytics do not require AI access.

### Harvester cannot find a column

Check the exact spelling of the column passed to `--col`. For Excel files, verify that the intended worksheet and header row are correct.
